Prototipo

Bienvenido, en este apartado se dara un acercamento sobre la relacion de la herramienta seleccionada para el reporte de cobertura de codigo y el prototipo de software propuesto

Basado en un proceso de benchmarkingbenchmarking es un proceso de crear, recopilar, comparar y analizar indicadores claves que permitan medir el rendimiento de los procesos las herramientas seleccionada segun la arquitectura del prototipo propuesta seria

Jacoco

Prototipo

El proceso de cobertura de codigo depende de la ejecución de pruebas unitarias Podemos decir que jacoco se integra junto con Junit,Powermock y mockito dependencias encargadas de las pruebas unitarias Jacoco indica al desarrollador el porcentaje de lineas cubiertas en pruebas unitarias La continuidad de los procesos DevOpsGrupo de prácticas que unifica operaciones de TI y desarrollo de software pueden restringirse segun el porcentaje de lineas cubiertas La integración de esta herramienta es una de sus mayores ventajas, se puede implementar en proyectos nuevos y en proyectos con mucho mas tiempo de ejecución Cuando se ejecuta un comando mvn que evite hacer un skip de las pruebas unitarias, las dependencias de Jacoco de forma automática van a generar un reporte de cobertura de código, el cual ofrece la vista HTML para brindar detalles A continuacion se muestra un ejemplo de como agregar la depencencia de Jacoco en el archivo de maven con extension .pom
