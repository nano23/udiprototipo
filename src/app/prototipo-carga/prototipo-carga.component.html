Jmeter/Blazemeter/Taurus

Prototipo

Para lograr determinar si los programas soportan cierto nivel de estres y disponibilidad, es importante ejecutar pruebas de Carga Existen programas que permiten realizar pruebas de carga de forma automática, para este caso del proyecto se utilizaran tres herramientas que son Jmeter,Blazemeter y Taurus BlazeMeter cuenta con una acción de grabar, en ella, se capturan procesos en la UI y procesos en la JMX, se pueden renombrar acciones para dividirlas en procesos Capturados los procesos en blazemeter, se deben terminar la ejecución del BlazeMeter, esto se logra a través del botón rojo Stop Finalmente en blazemeter se deben descargar los archivos scriptssecuencia de comandos en un lenguaje legible por una maquina que fueron obtenidos del BlazeMeter, y seleccionar con que herramienta será compatible, para el caso del proyecto prototipo web, serán descargados para Jmeter (JMX) A continuacion se evidencia un ejemplo de script jmx generado desde blazemeter:

En Jmeter es posible importar los scriptssecuencia de comandos en un lenguaje legible por una maquina generados por Blazemeter, esto con la finalidad de realizar modificaciones, parametrizaciones que permitan ejecutar una prueba masiva de forma exitosa Jmeter es una herramienta potente por si sola, y tambien permite generar reportes, para esto se pulsa sobre el Thred Group y se selecciona el reporte deseado, entre ellos, reporte grafico con medición en milisegundos o reporte de tiempo, entre otros Usando Taurus, se procede a ejecutar el archivo PROTOTIPO-DEVOPS-UDI.jmx que fue el obtenido del Jmeter despues de hacerle modificaciones proveniente del Blazemeter Para ejecutarlo desde Taurus primero ponemos bzt mas el nombre del archivo, ejemplo, 'bzt' PROTOTIPO-DEVOPS-UDI.jmx, lo cual generara un archivo con extension yml Finalmente el archivo yml es el encargado de ejecutar las pruebas de carga completas, este archivo se enlaza con blazedemo que es un servicio que posee BlazeMeter en ambientes de pruebas A continuacion se evidencia un ejemplo de script yml generado desde Taurus:

Script de prueba generado por Taurus: prototipo.yml


                    execution:
                    - concurrency: 1000
                      hold-for: 1m
                      ramp-up: 10s
                      #iterations: 1
                      #throughput: 10
                      #steps: 10
                      scenario: Thread Group
                    scenarios:
                      Thread Group:
                        requests:
                        - label: blazedemo
                          method: GET
                          url: http://blazedemo.com/
                        script: PROTOTIPO-DEVOPS-UDI.jmx
