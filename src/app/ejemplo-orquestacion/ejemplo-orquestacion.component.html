Ejemplo de orquestación con Jenkins

En jenkins es posible visualizar por consola lo sucedido en la ejecucion de procesos automaticos, esto con la finalidad, de conocer el origen de un problema en caso de que se de presente algun fallo y/o bloqueo Instalacion Dependencias Maven Servidor: permite comprobar la instalación de dependencias Maven del servidor, la función de esta tarea es copiar las dependencias del Maven en el entorno de Jenkins Pruebas Unitarias Servidor y Pruebas Unitarias Cliente : en esta etapa se realiza la ejecución de pruebas unitarias del lado del cliente y del lado del servidor Compilación Cliente:etapa en la que se ejecuta comando para realizar proceso de compilación del lado del cliente, comando usado, npm run ng build Cargar dependencia Proyecto - Nexus: tarea para el despliegue en el servidor Nexus de copia de codigo fuente en donde está el repositorio de artefactos del proyecto Ejecutar Reporte SonarQube: tarea para ejecutar los reportes de SonarQube, el comando ejecutado es mvn sonar:sonar Copiar a pruebas Servidor: Se realiza una tarea para poder copiar al servidor de pruebas el codigo compilado Subir aplicación Cliente: la funcionalidad es validar que el puerto este habilitado, para posteriormente ejecutar el comando de despliege en nodejsentorno en tiempo de ejecución multiplataforma, de código abierto en el lenguaje javascript Subir aplicación Servidor: tarea que permite subir la aplicación del lado del servidor, la funcionalidad es validar que el puerto este habilitado, y en caso contrario continuar con el despliegue satisfactorio Pruebas de Integración:tarea de pruebas de integración dependen de que ya la aplicación está montada, ya que su funcionalidad es ejecutar el JSONes un formato de texto sencillo para el intercambio de datos de postman por medio de newman Pruebas de Carga:tarea de pruebas de carga ejecutata con el comando btz prototipo.yml -report Correo Confirmación de pruebas:Como ultima tarea lo que hace Jenkins es enviar un correo electrónico por medio de la función mail que viene dentro de los plugins instalados por default y notificar el estado de ejecucion del pipeline
